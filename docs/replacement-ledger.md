## Production Replacement Priority Backlog

This section is the high-level replacement queue. The detailed tables above remain the source of truth for each area, but these items should be checked before every production-readiness milestone.

1. Storage and migrations: replace SQLite-only team paths with `PostgresAgentStore`, versioned migrations, tenant isolation tests, backup/restore guidance, and compatibility tests that keep SQLite local mode.
2. Auth and policy: replace local tokens and hard-coded role maps with authenticated users, groups, service accounts, agent sessions, role templates, revocation, quorum approval, and policy-service-backed RBAC/ABAC.
3. Broker and events: replace local assignment polling with Redis/NATS/Postgres queues, event streams, visibility timeouts, delayed retries, poison queues, metrics, and distributed nonce replay windows.
4. Workers and remote agents: replace foreground CLI loops with supervised daemons, mandatory signed heartbeat/lease verification, key rotation, revocation, stale recovery, health streaming, graceful drain/shutdown, and distributed delivery cursors.
5. Workspace execution: replace Node shell/file execution with Rust/container runner, robust patch engine, resource limits, network policy, sandbox options, artifact capture, and teardown.
6. Secrets: replace passphrase file vault for shared deployments with OS keychain locally and Vault/KMS/enterprise secret managers in production, including short-lived leases, rotation, revocation, and scoped delivery.
7. Artifacts and retention: replace inline/local artifact handling with object storage, lifecycle workers, legal hold, immutable manifests, signed deletion/export bundles, and SIEM sinks.
8. MCP and plugins: harden the local MCP execution path with sandboxed client transports, Web/API diagnostics, agent-tool grants, signed approval envelopes, quorum continuation, signed plugin packages, persistent capability grants, and plugin isolation.
9. RAG and memory: replace keyword retrieval with ACL-filtered hybrid full-text/vector search, embedding providers, rerankers, citation precision checks, permission-leak eval, freshness checks, signed eval reports, and CI trend gates.
10. Git automation: replace dry-run URL/local Git helpers with GitHub App/GitLab OAuth adapters, stored PR/MR refs, isolated worktrees, webhook/CI ingestion, and repair iteration policy.
11. UI and operations: replace the local token-gated console with authenticated TUI/Web UI surfaces, real-time room/task/spec timelines, admin retention/export controls, observability, deployment packaging, upgrades, backups, and runbooks.
